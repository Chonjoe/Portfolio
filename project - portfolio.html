Personal Portfolio Website

About This Project

My first hand-coded website, a 2-page developer portfolio built from scratch with HTML and CSS. This site features a responsive layout, project grid with cards, a contact form with proper labels, and semantic HTML structure. No frameworks, no templates, no copy-paste. Just raw HTML and CSS written line by line.

What I Used

  • HTML5 for semantic structure (header, nav, main, section, article, footer)
  • CSS3 with Flexbox for navigation layout
  • CSS Grid for the responsive project card layout
  • CSS Variables for consistent theming and colors
  • Media queries for mobile responsiveness
  • No JavaScript — pure HTML/CSS portfolio

What I Learned

  • Semantic HTML matters: Using <header>, <nav>, <main>, <section>, <article>, and <footer> made my code readable and accessible — not just a bunch of <div>s.
  • Flexbox for navigation: `display: flex` + `justify-content: space-between` finally made my logo stay left and links stay right. That single line was a breakthrough.
  • CSS Grid for cards: `display: grid` with `grid-template-columns: repeat(3, 1fr)` turned three messy cards into a clean, professional grid.
  • Images don't resize themselves: My profile photo was huge until I added `width: 100%; height: auto; object-fit: cover;` — now it's a clean circle on all devices.
  • Mobile responsiveness is a choice: One `@media` query transforms the 3-column grid into 1 column on phones. Without it, my portfolio breaks on the device most people use.
  • CSS variables save hours: Defining `--primary-color` once and reusing it everywhere made changing my entire color scheme a one-line edit.
  • Building > watching: I watched CSS tutorials for weeks. Nothing clicked until I built this page, broke things, and fixed them myself.
